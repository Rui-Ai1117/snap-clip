Chrome Extension — Privacy Policy
Privacy Policy
Last updated: May 9, 2026

🔒 Summary

SnapClip does not collect, store, transmit, or share any personal data whatsoever. All operations are performed entirely within your browser on your local device.

✓ No data collection
✓ No external servers
✓ No tracking or analytics
✓ No account required

📋 Data We Collect

SnapClip collects no personal data. Specifically:

  • Screenshots are captured in memory only and never written to disk by the extension
  • Clipboard data is written only when you explicitly press the Copy button
  • No browsing history, URLs, or page content is recorded
  • No user identifiers or device identifiers are collected

🛠 Permissions Used

SnapClip requests the following Chrome permissions, each used solely to provide core functionality:

  • activeTab — captures the currently visible tab when you initiate a screenshot
  • scripting — injects the overlay UI into the current tab so the panel appears on-page
  • tabs — reads the current window ID required by the capture API

No permission is used to access data on pages you visit, track navigation, or communicate with external services.

🌐 Third-Party Services

SnapClip does not integrate with, send data to, or load resources from any third-party service or server. The extension operates entirely offline after installation.

🔄 Changes to This Policy

If this policy changes, the updated version will be published on this page with a revised date. Continued use of the extension after changes constitutes acceptance of the updated policy.

✉️ Contact

If you have questions about this privacy policy or the extension, please open an issue on the extension's support page or contact the developer directly.
